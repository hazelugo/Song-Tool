- Created POST /api/discovery route (src/app/api/discovery/route.ts):
  - Parses natural language prompt: time sig → key sig → musical key → explicit BPM → mood keywords → leftover as searchTerm
  - Mood keyword → BPM mapping: upbeat/fast/energetic/dance/workout → bpmMin:120; slow/ballad/chill → bpmMax:90
  - searchTerm searches: name ILIKE, lyrics FTS (websearch_to_tsquery), tags EXISTS
  - Tags: split searchTerm into individual words, one EXISTS subquery per word (so "acoustic ballad" matches songs tagged "acoustic" OR "ballad")
