# Song Tool — Impeccable Design Context

## Design Context

### Users
Working musicians and performers actively managing a live song catalog. Their context: booking gigs, building setlists, prepping for rehearsal — often under time pressure. They need to find the right song fast, not browse leisurely. The interface should respect their time and feel like a professional tool, not a consumer app.

### Brand Personality
**Precise. Purposeful. Dark.**

Like a DAW opened at 2am in a recording studio — functional, serious, and quietly satisfying to use. Not flashy or decorative. The music is the content; the UI recedes.

Reference: Ableton Live, Logic Pro — dense but organized, respects user intelligence, rewards familiarity.

Anti-reference: Streaming apps (Spotify consumer aesthetic), playful SaaS (rounded bubbles, pastel gradients, marketing-speak).

### Aesthetic Direction
- **Dark-mode primary**: Dark should feel native and considered, not an afterthought
- **Light-mode parity**: Light mode gets equal polish — not a washed-out afterthought
- **Neutral OKLCH palette**: Already in place — no gratuitous color; color used for meaning (status, focus, destructive)
- **Dense but scannable**: Tabular layouts, monospaced data (BPM, key, time sig), clear typographic hierarchy
- **Minimal chrome**: Navigation stays out of the way; content fills the space
- **Geist type system**: Sans for UI, Mono for musical data values

### Design Principles

1. **Function earns every pixel** — No decorative elements that don't carry information or aid interaction. If it doesn't help a gigging musician find the right song faster, it doesn't belong.

2. **Scan speed over visual delight** — A user under performance pressure needs to read BPM, key, and tags at a glance. Typography weight, spacing, and contrast should optimize for rapid scanning, not aesthetic exploration.

3. **Dark-native, light-consistent** — Dark mode is the spiritual home of this tool. Design dark first; translate to light without losing density or character.

4. **Music data is the hero** — Musical attributes (key, BPM, time signature, chord progressions) should be typographically prominent. They are the reason this tool exists.

5. **Professional tool, not product** — Favor the Ableton/Logic aesthetic: structured, information-dense, keyboard-friendly, zero marketing tone. Avoid consumer softness (rounded-everything, hover animations for decoration, empty-state mascots).
